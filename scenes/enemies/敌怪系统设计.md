# 敌怪系统设计

## 设计目标

- **组件化**：移动 / 检测 / 攻击 / 状态机各自独立为可替换组件
- **数据驱动**：每种敌怪通过 `EnemyConfig` (Resource) 配置参数，无需写代码即可创建新敌怪
- **复用现有**：GodotEnemy 的寻路避障逻辑保留，抽取为 `EnemyMovement` 组件
- **扩展性强**：新攻击类型只需新建 `EnemyAttack` 子类

---

## 架构总览

```
EnemyConfig (.tres)          ← 数据：行为/速度/伤害/攻击类型
		│
		▼
BaseEnemy (CharacterBody2D)  ← 组件容器，_ready() 组装所有组件
├── EnemyStateMachine         ← 状态驱动：IDLE → CHASE → ATTACK → CHASE
├── EnemyMovement             ← 移动：寻路/避障/游走/后撤/冲锋
├── EnemyDetection            ← 检测：半径检测玩家，信号通知
├── EnemyAttack (子类)         ← 攻击：近战/远程/接触
├── HitboxComponent           ← 攻击判定框
├── HurtboxComponent          ← 受击判定框
└── HealthComponent           ← 血量
```

### 状态流转

```
IDLE (随机游走)
  │ player_detected
  ▼
CHASE (朝玩家移动)
  │ 进入攻击范围
  ▼
ATTACK (执行攻击)
  │ attack_finished 或 超出范围
  ▼
CHASE ...

player_lost → IDLE
```

## 文件清单

### 组件 (`scenes/enemies/components/`)

| 文件 | 类 | 职责 |
|------|-----|------|
| `enemy_state_machine.gd` | EnemyStateMachine | 状态机：IDLE/CHASE/ATTACK/FLEE/STORED |
| `enemy_movement.gd` | EnemyMovement | 移动：寻路 + 扇形射线避障 + 卡墙脱困 + 游走/后撤/冲锋 |
| `enemy_detection.gd` | EnemyDetection | 玩家检测：半径检测 + `player_detected`/`player_lost` 信号 |
| `enemy_attack.gd` | EnemyAttack | 攻击基类：冷却管理 + `attack_started`/`attack_finished` 信号 |
| `enemy_attack_melee.gd` | EnemyAttackMelee | 近战：短暂激活 HitboxComponent |
| `enemy_attack_ranged.gd` | EnemyAttackRanged | 远程：调用 BulletFactory.spawn() |
| `enemy_attack_contact.gd` | EnemyAttackContact | 接触：body_entered 时直接对玩家 Hurtbox 造成伤害 |

### 数据与工厂

| 文件 | 类 | 职责 |
|------|-----|------|
| `enemy_config.gd` | EnemyConfig | Resource：敌怪参数（速度/伤害/行为/攻击类型） |
| `enemy_factory.gd` | EnemyFactory | 便捷生成：`spawn(config, position)` |
| `EnemyBase.tscn` | — | 组件骨架场景，挂载所有组件节点 |

### 敌怪基类

| 文件 | 类 | 职责 |
|------|-----|------|
| `base_enemy.gd` | BaseEnemy | 组件容器：_ready() 根据 config 组装/配置各组件 |
| `godot/godot_enemy.gd` | GodotEnemy | 扩展：编辑器扇形可视化（`@tool`） |

## 扩展指南

### 添加新小怪类型（已有攻击模式）

1. 复制 `configs/` 中的一个 `.tres`，修改参数
2. 通过 `EnemyFactory.spawn(config, pos)` 生成

### 添加新攻击类型

1. 新建脚本 `extends EnemyAttack`
2. 实现 `_perform_attack(target)`，结束时调用 `attack_finished.emit()`
3. 在 `EnemyConfig.AttackType` 枚举中添加新类型
4. 在 `EnemyBase.tscn` 中添加组件节点
5. 在 `base_enemy.gd` 的 `_apply_config()` 中添加配置逻辑

### 添加新行为模式

1. 在 `EnemyConfig.Behavior` 枚举中添加
2. 在 `EnemyStateMachine._process()` 中添加对应逻辑
